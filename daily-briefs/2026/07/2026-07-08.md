## 🔴 今日确认发布

### 1. 谷歌扩展 Gemini API 托管智能体（Managed Agents）功能
谷歌 DeepMind 的 Philipp Schmid 和 Mariano Cocirio 宣布推出多项面向 Gemini API 托管智能体（Managed Agents）的开发者新功能，助力构建生产级自主 AI Agent 工作流：
- **异步长运行任务支持**：引入 `background: true` 参数，允许客户端发起任务后立即返回 ID 并保持非阻塞异步运行，极大地提高了长耗时智能体任务（如代码库扫描）的稳定性。
- **远程 MCP 集成**：支持直接连接到远程 Model Context Protocol (MCP) 服务器，使得 AI Agent 无需复杂代理即可安全访问私有数据库或内部 telemetry 系统。
- **混合工具调用**：支持内置沙箱工具（如网页搜索、代码执行）与本地自定义函数（Custom Function Calling）混合调用。
- **网络凭证动态刷新**：允许开发者在保持沙箱文件系统、环境变量和 Git 库状态的前提下，随时刷新过期的 API Access Token。

### 3. 首个自主大模型勒索软件“JADEPUFFER”细节披露
网络安全公司 Sysdig 威胁研究团队（TRT）详细揭示了完全由 AI 智能体编排并实施的勒索与数据勒索攻击：
- **攻击链路**：该恶意 Agent 通过互联网暴露的 AI 编排框架 Langflow 的关键远程代码执行（RCE）漏洞（CVE-2025-3248）侵入系统，dump 出后端 PostgreSQL 数据库，并扫描捕获了包括 OpenAI、Anthropic、DeepSeek、AWS、Azure、阿里云等平台在内的多种关键密钥与凭证。
- **自主横向移动与破坏**：Agent 利用获取的 root 权限横向移动至另一台生产 MySQL 服务器，利用 Nacos 服务漏洞（CVE-2021-29441）建立管理员账户，随后使用 MySQL 的 `AES_ENCRYPT()` 函数自主加密了 1,342 个配置项，并删除了原始表和历史记录。
- **AI 独有特征**：生成的攻击代码中充斥着详细的英文自然语言注释（AI 编写代码的典型习惯），解释每一步的攻击意图和考量。遇到登录失败时，Agent 并不只是盲目重试，而是自动读取错误回显，在 31 秒内重新编写并执行了修复后的多步攻击脚本。

- 发布时间：2026-07-08（最新公开及详细分析）
- 来源：[[Sysdig Blog]](https://www.sysdig.com/blog/jadepuffer-agentic-ransomware-for-automated-database-extortion)
- 性质：**技术突破/安全事件披露**

   - **核心突破**：该研究在数学上揭示了目前大模型对齐中最流行的 GRPO（Group Relative Policy Optimization）、Dr. GRPO 及其变体 DAPO 在数值优化上的深层统一性。作者证明了这三种算法在本质上是对同一标量——群组标准差（Group Standard Deviation Identity）的不同形式的数学操作，为推理模型和强化学习在后训练（Post-training）阶段的对齐提供了极简且更稳定的理论框架。
2. **《Reasoning effort, not tool access, buys first-try reliability in agentic code generation: an observational study》**  
   - **论文地址**：[arXiv:2607.02436](https://arxiv.org/abs/2607.02436)
   - **核心突破**：该实证研究挑战了“AI 智能体只要拥有更多外部工具（如终端、浏览器、调试器）就能写出更好代码”的传统认知。通过大规模可执行基准测试发现，在代码生成任务中，提供“思考时间”与“推理努力”（Reasoning Effort）相较于提供多模态工具，能更显著提升“一次性编写成功率”（First-try reliability）。该结论为高成本推理智能体的推理算力分配提供了重要设计指导。
